## Goal

- Ship a **standalone** exploration-strategy skill (`java-codebase-explore`) as
  canonical markdown plus a **Perplexity-format** `.zip`, both regenerated by
  one repo script so they never drift.
- Keep **`docs/AGENT-GUIDE.md`** as the operating manual (how / shapes /
  recovery); the new artefact is the **strategy guide** (when / why / missions /
  fallbacks / anti-capabilities / stopping rules).
- Integrate discoverability via **README** pointers next to existing
  AGENT-GUIDE mentions (§3 agent bullets + §9 further-reading table).
- **No** Python/MCP/schema changes; **no** `ontology_version` bump for this
  effort (cheat sheet must still stay in lockstep with ontology bumps **when**
  they happen elsewhere).

## Principles (do not relitigate in review)

- **Strategy not surface:** no full argument-shape duplication; cheat sheet
  only; link to `docs/AGENT-GUIDE.md` for shapes, recovery, slash aliases.
- **Standalone operability:** agent can execute missions with only the skill
  body + live MCP tool descriptions; minimum taxonomy inlined in the cheat
  sheet appendix only.
- **MCP is one tool among several:** `rg`, file reads, `git log`, README/build
  files, and `java-codebase-rag` CLI are first-class fallbacks with explicit
  rules of thumb.
- **Missions before tools:** top-level narrative is mission-scoped; six missions
  only (count locked in propose §7 item 5; names in §3.3); a seventh mission
  requires a new propose, not a drive-by PR.
- **Stopping rules are mandatory** per mission; anti-capabilities are their
  own section (not a footnote); Appendix A ships as a **seven-item** enumerated
  list in the skill — new items need a propose amendment (cap mirrors propose §8
  risk row).
- **Section order is fixed** (propose §3.2 outline; same order locked in §7
  item 6): Activation → Pre-flight → Map the seams → Mission catalogue → When
  MCP is the wrong layer → What this MCP is NOT → Confidence and staleness →
  Anti-patterns → Cheat sheet appendix.
- **English-only** skill body; **Perplexity v1 packaging only** (no Claude Code
  / Cursor skill bundles until a downstream consumer asks).
- **Target size:** skill body ≤ ~800 lines including appendix (propose §8).
